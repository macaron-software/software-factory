# Architecture Design

## Purpose
Design scalable, maintainable, and resilient system architectures.

## Principles
- Favor simplicity over cleverness
- Design for change and extensibility
- Apply separation of concerns
- Document architectural decisions (ADRs)
- Consider non-functional requirements early

## Practices
- Create clear component boundaries and interfaces
- Use established patterns (hexagonal, event-driven, CQRS) where appropriate
- Plan for observability (logging, metrics, tracing)
- Review architecture with the team before implementation
- Evaluate trade-offs explicitly
