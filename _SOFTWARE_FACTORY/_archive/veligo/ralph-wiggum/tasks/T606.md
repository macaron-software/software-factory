# Task T606: Fix owner login - créer utilisateur owner

**Priority**: P0
**WSJF Score**: 9.0
**Queue**: TDD
**Domain**: api

## Description
La page owner.veligo.app affiche des identifiants par défaut:
`owner@veligo.app / owner123!`

Mais le login échoue avec "Email ou mot de passe incorrect".

## Cause probable
L'utilisateur owner n'existe pas dans la base de données,
ou le hash du mot de passe ne correspond pas.

## Solution
1. Créer migration pour insérer utilisateur owner
2. Hash bcrypt du mot de passe owner123!
3. Role = OWNER dans la table users

## Migration SQL
```sql
INSERT INTO users (id, email, password_hash, role, tenant_id, created_at)
VALUES (
  gen_random_uuid(),
  'owner@veligo.app',
  '$2b$12$...', -- bcrypt hash de owner123!
  'OWNER',
  NULL, -- owner n'a pas de tenant spécifique
  NOW()
)
ON CONFLICT (email) DO NOTHING;
```

## Test E2E
```typescript
test('owner can login with default credentials', async ({ page }) => {
  await page.goto('https://owner.veligo.app');
  await page.fill('[data-testid="owner-email"]', 'owner@veligo.app');
  await page.fill('[data-testid="owner-password"]', 'owner123!');
  await page.click('[data-testid="owner-login-submit"]');
  await expect(page).toHaveURL(/.*dashboard/);
});
```
