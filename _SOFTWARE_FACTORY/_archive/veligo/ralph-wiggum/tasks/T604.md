# Task T604: Fix pages blanches /signup et /stations

**Priority**: P0
**WSJF Score**: 10.0
**Queue**: TDD
**Domain**: frontend

## Description
Les pages /signup et /stations sont complètement blanches en production.
Le composant Svelte crash probablement à cause d'une erreur JS.

## Erreur console
```
SyntaxError: Unexpected token '<', "<!doctype html>" is not valid JSON
```

Le frontend reçoit du HTML au lieu de JSON → appel API qui retourne 404.

## Diagnostic
1. /signup appelle un endpoint qui n'existe pas
2. Le fallback SvelteKit retourne le HTML de la page
3. Le code essaie de parser ce HTML comme JSON → crash

## Solution
1. Identifier les appels API dans +page.svelte de /signup
2. Vérifier que les endpoints backend existent
3. Implémenter les endpoints manquants
4. Ajouter error handling côté frontend

## Fichiers à analyser
- `veligo-platform/frontend/src/routes/(app)/signup/+page.svelte`
- `veligo-platform/frontend/src/routes/(app)/stations/+page.svelte`
