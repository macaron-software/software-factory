# RTK Instructions for OpenCode

**IMPORTANT:** Use `rtk` prefix for ALL file/git/test commands to save 60-90% tokens.

## Command Mappings

Always use these instead of raw commands:

```bash
# Git commands
rtk git status          # instead of: git status
rtk git diff            # instead of: git diff
rtk git log -n 10       # instead of: git log
rtk git add .           # instead of: git add
rtk git commit -m "msg" # instead of: git commit
rtk git push            # instead of: git push

# File commands
rtk ls                  # instead of: ls
rtk read file.py        # instead of: cat file.py
rtk grep "pattern" .    # instead of: grep "pattern" .
rtk find "*.py" .       # instead of: find . -name "*.py"

# Test commands
rtk cargo test          # instead of: cargo test
rtk pytest              # instead of: pytest
rtk vitest run          # instead of: vitest run

# Lint commands
rtk ruff check          # instead of: ruff check
rtk lint                # instead of: npm run lint

# Docker
rtk docker ps           # instead of: docker ps
rtk docker logs <id>    # instead of: docker logs <id>
```

## Why RTK?

- **Token savings:** 60-90% reduction on command outputs
- **Same functionality:** Commands work identically, output is just optimized
- **Current stats:** 19.2M tokens saved (93.5% efficiency)

## Verify RTK is Working

```bash
rtk --version   # Should show 0.24.0 or higher
rtk gain        # Show savings statistics
```

## Examples

❌ **Don't do this:**
```bash
git status
# Outputs 150+ lines of verbose text
```

✅ **Do this instead:**
```bash
rtk git status
# Outputs 10 lines of compact, essential info with 📌 emoji
```

## Check Current Savings

```bash
rtk gain --history    # Recent commands
rtk gain --daily      # Daily breakdown
```

**Remember:** RTK is already integrated in Claude Code (automatic) and configured for Copilot (shell functions). For OpenCode, you must explicitly use the `rtk` prefix.
