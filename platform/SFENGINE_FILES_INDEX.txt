================================================================================
CRITICAL FACTS FOR NEW PATTERNS
================================================================================

1. ALWAYS use &[&str] for agent_ids
   → Never Vec<String>
   → Always check agents::get_agent() returns Some()

2. ALWAYS use Result<String, String>
   → No custom error types
   → Convert with .into()

3. ALWAYS call executor::run_agent() with protocol
   → protocol = protocols::protocol_for_role(&agent.role, phase)
   → Pass as Some(protocol)

4. ALWAYS call guard::check_l0() after execution
   → Even if you ignore result
   → Emit Response if guard fails

5. ALWAYS emit events
   → AgentEvent::Thinking before work
   → AgentEvent::Response for all results
   → Optional: AgentEvent::Error for failures

6. ALWAYS strip emoji
   → content = strip_emoji(&result.content.unwrap_or_default())

7. ALWAYS store messages
   → store_agent_msg() or store_discussion_msg()

8. ALWAYS handle empty agent_ids
   → return Err("No agents found".into())

9. ALWAYS use French in prompts
   → STYLE_RULES enforces French formatting
   → Append STYLE_RULES to every system prompt

10. ALWAYS join outputs with separator
    → outputs.join("\n\n---\n\n")

================================================================================
AGENTEVENTS YOU NEED TO KNOW
================================================================================

AgentEvent::Thinking
   → Status: agent is thinking
   → Usage: on_event(&agent.id, AgentEvent::Thinking)

AgentEvent::Response { content: String }
   → MOST USED: final text, JSON, or status
   → Usage: on_event(&agent.id, AgentEvent::Response { 
            content: "text or JSON".into() })

AgentEvent::ToolCall { tool: String, args: String }
   → Tool being invoked (executor only)

AgentEvent::ToolResult { tool: String, result: String }
   → Tool result returned (executor only)

AgentEvent::Error { message: String }
   → Error condition

AgentEvent::Reasoning { active: bool }
   → Extended thinking indicator (o1 models)

AgentEvent::ResponseChunk { content: String }
   → Streaming chunk (executor only)
